# ACP/A2A Comparison

`llm-collab` is intentionally a local, file-based coordination workspace.

## Why it differs

- No always-on server runtime
- No network transport dependency
- Git-friendly, human-readable artifacts
- Works offline and inside existing repo workflows

## What ACP/A2A provide

- Standardized agent protocol surface
- Interoperable remote transport contracts
- Runtime-to-runtime messaging semantics

## Practical position

Use `llm-collab` when you need lightweight, local-first collaboration with transparent state files.
Use ACP/A2A when your architecture needs protocol-level interoperability across distributed runtimes.
